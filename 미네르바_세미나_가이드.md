# 🎓 미네르바 스타일 AI 세미나 가이드

> **"하나의 아이디어를 다양한 렌즈로 보는 통찰력"**
> 
> 미네르바 스쿨의 Active Learning + AI 협업으로  
> **보고 → 느끼고 → 사용하고 → 만들고 → 수정하는** 실전 학습

---

## 🎯 미네르바 스쿨이란?

### 세계에서 가장 혁신적인 대학

```mermaid
mindmap
  root((미네르바 스쿨))
    Active Learning
      교수 강의 NO
      학생 주도 토론
      실시간 참여
    다각도 사고
      Multiple Lenses
      학제간 접근
      통합적 이해
    실전 학습
      7개국 이동
      실제 프로젝트
      현장 경험
    소크라테스식
      끊임없는 질문
      비판적 사고
      논리 구조화
```

### 미네르바의 핵심 원칙

| 원칙 | 설명 | 기존 교육과의 차이 |
|------|------|------------------|
| **No Lectures** | 교수 강의 없음 | 학생이 먼저 학습 → 세미나에서 토론 |
| **Active Learning** | 100% 참여형 수업 | 듣기만 하는 게 아니라 말하고 토론 |
| **Multiple Perspectives** | 하나의 주제를 여러 렌즈로 | 경제/윤리/심리/역사/기술 등 다각도 분석 |
| **Real-world Problems** | 실제 문제 해결 | 이론이 아니라 실전 프로젝트 |
| **Seminar-style** | 15-20명 소규모 토론 | 대형 강의 NO |
| **Flipped Classroom** | 거꾸로 교실 | 집에서 학습 → 수업에서 토론 |

### 10가지 사고 렌즈 프레임워크

| 렌즈 | 핵심 질문 | 예시 (AI 챗봇 도입) |
|------|----------|---------------------|

| **🌍 환경** | 환경 영향은? 지속가능성은? | • 서버 에너지 소비<br/>• 탄소 배출<br/>• 전자 폐기물 |
| **🏛️ 정치** | 권력 관계는? 규제는? | • 빅테크 독점<br/>• 정부 규제 필요성<br/>• 국제 경쟁 |
| **🎨 문화** | 문화적 의미는? 가치관 변화는? | • 창작 문화 변화<br/>• 저작권 개념<br/>• 인간성 재정의 |
| **🔮 미래** | 10년 후는? 시나리오는? | • 완전 자동화 사회<br/>• 베이직 인컴<br/>• 새로운 직업 |

## 🎓 미네르바 스타일 세미나 운영법

### 세미나 구조 (2시간)

```mermaid
gantt
    title 미네르바 세미나 타임라인
    dateFormat HH:mm
    section 사전 학습
    개인 학습 (집에서)     :done, 00:00, 12:00
    section 세미나

    통합 토론              :11:20, 20min
    AI 협업 실습           :11:40, 15min
    마무리 & 과제          :11:55, 5min
```

### 1단계: 사전 학습 (Flipped Classroom)

**세미나 3일 전 공지**:

```markdown
📚 다음 세미나 준비

[주제]
━━━━━━━━━━━━━━━━━━━━━━━━
"ChatGPT가 교육에 미치는 영향"
(또는 독서 주제)

[사전 학습 과제 (3일간)]
━━━━━━━━━━━━━━━━━━━━━━━━
□ 필독 자료 읽기
  • 교재: ○○○ (Chapter 3)
  • 논문: "AI in Education" 
  • 기사: 3개 (링크 제공)

□ 10가지 렌즈로 분석하기
  각 렌즈별로 핵심 질문 1개씩 준비
  (템플릿 제공)

□ AI와 대화하기
  ChatGPT에게 각 렌즈 관점에서 질문
  흥미로운 답변 3개 캡처

□ 나의 입장 정리
  찬성/반대/유보 + 근거 3가지

[제출]
━━━━━━━━━━━━━━━━━━━━━━━━
세미나 1일 전까지 Notion에 업로드
(최소 500자)
```

### 2단계: 세미나 진행

#### 파트 1: 체크인 & 컨텍스트 (10분)

```markdown
🎬 세미나 시작

[체크인 라운드 (5분)]
━━━━━━━━━━━━━━━━━━━━━━━━
한 사람씩 돌아가며 (각 30초):
"사전 학습하며 가장 놀라웠던 점 한 가지"

[오늘의 목표 (2분)]
━━━━━━━━━━━━━━━━━━━━━━━━
퍼실리테이터:
"오늘은 [주제]를 10가지 렌즈로 분석합니다.
목표는 **하나의 답**을 찾는 게 아니라,
**복잡성을 이해**하고 **통찰을 발견**하는 것입니다."

[Ground Rules 확인 (3분)]
━━━━━━━━━━━━━━━━━━━━━━━━
□ 모든 의견은 존중된다
□ 질문은 판단이 아니라 호기심
□ 듣기 50%, 말하기 50%
□ "예/아니오"가 아니라 "왜?"
□ AI를 적극 활용 (실시간 검증)
```

#### 파트 2: 렌즈별 심층 토론 (60분)

**토론 방식: 소크라테스 메소드**

```markdown
🔍 렌즈 1: 경제적 관점 (10분)

[퍼실리테이터 시작 질문]
━━━━━━━━━━━━━━━━━━━━━━━━
"ChatGPT가 교육에 도입되면 
누가 경제적 이익을 얻고, 누가 손해를 보나요?"

[소크라테스식 대화]
━━━━━━━━━━━━━━━━━━━━━━━━
학생 A: "학생들이 이익을 봅니다. 무료니까요."
퍼실리테이터: "정말 무료인가요? 숨은 비용은?"
학생 B: "데이터와 시간이 비용이죠."
퍼실리테이터: "좋아요. 그럼 학교는 어떻게 되나요?"
학생 C: "선생님이 줄어들 수 있습니다."
퍼실리테이터: "줄어든다면 어떤 선생님이 먼저?"
학생 D: "단순 지식 전달하는 선생님요."
퍼실리테이터: "그럼 남는 선생님은 어떤 역할?"
...

[AI 실시간 활용]
━━━━━━━━━━━━━━━━━━━━━━━━
학생: "실제 데이터가 있나요?"
→ 즉시 ChatGPT에게 질문
→ 답변 공유 및 검증
→ 토론 심화

[정리 (2분)]
━━━━━━━━━━━━━━━━━━━━━━━━
• 핵심 인사이트 3가지
• 추가 질문 1개
• 다음 렌즈와의 연결점
```

**각 렌즈별 10분씩 진행** (총 6개 렌즈)

## 🔄 5단계 학습 사이클: 보고-느끼고-사용하고-만들고-수정

### 완전한 학습 루프

```mermaid
graph LR
    A[1.보기<br/>SEE] --> B[2.느끼기<br/>FEEL]
    B --> C[3.사용하기<br/>USE]
    C --> D[4.만들기<br/>MAKE]
    D --> E[5.수정하기<br/>REVISE]
    E --> F[통찰 & 성장]
    F --> A
    
    A --> A1[다양한 예시<br/>다각도 관찰<br/>패턴 인식]
    B --> B1[감정 인식<br/>직관 개발<br/>공감 형성]
    C --> C1[실제 적용<br/>실험<br/>테스트]
    D --> D1[창조<br/>구현<br/>프로토타입]
    E --> E1[피드백<br/>개선<br/>반복]
    
    style A fill:#e1f5ff,color:#111
    style F fill:#ccffcc,color:#111
```

### 각 단계별 활동

#### 1. 보기 (SEE) - 많이 보고

**목표**: 다양한 관점과 예시로 아이디어 이해하기

```markdown
📺 보기 단계 활동

[다양한 소스에서 보기]
━━━━━━━━━━━━━━━━━━━━━━━━
□ 책/논문 읽기
□ 사례 연구 분석
□ 전문가 인터뷰 시청
□ 경쟁 제품 분석
□ 역사적 사례 조사

[10가지 렌즈로 보기]
━━━━━━━━━━━━━━━━━━━━━━━━
각 렌즈마다:
• 사례 1개씩 수집
• 패턴 발견
• 공통점/차이점

[AI 활용]
━━━━━━━━━━━━━━━━━━━━━━━━
ChatGPT 프롬프트:
"[주제]에 대해 10가지 다른 관점에서
각각 대표적인 사례를 하나씩 보여줘.
각 사례마다 배울 점을 설명해줘."
```

**예시: "배달 앱" 아이디어를 보기**

| 렌즈 | 기존 사례 | 관찰 포인트 |
|------|----------|-----------|
| 경제 | 배달의민족, 우버이츠 | 수수료 구조, 시장 점유율 |
| 기술 | 알고리즘 매칭 | 배차 최적화 방식 |
| 사회 | 배달 라이더 | 노동 조건, 사회적 논란 |
| 환경 | 일회용 용기 | 플라스틱 문제 |
| ... | ... | ... |

#### 2. 느끼기 (FEEL) - 느끼고

**목표**: 직관과 감정으로 깊이 이해하기

```markdown
💭 느끼기 단계 활동

[감정 매핑]
━━━━━━━━━━━━━━━━━━━━━━━━
각 이해관계자의 감정:
• 사용자: 편리함 vs 의존
• 라이더: 자유 vs 불안
• 식당: 매출 증가 vs 수수료 부담
• 환경: 죄책감

[공감 인터뷰]
━━━━━━━━━━━━━━━━━━━━━━━━
```
